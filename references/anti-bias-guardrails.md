# 反偏见硬约束与自检清单

> 多队对抗的最大威胁是"看起来在辩论，实际在从众"，以及"法官偏袒自己的方案"。本文沉淀反偏见硬约束 + 法官自检清单。

---

## 一、7 条反偏见硬约束（必须遵守）

| # | 约束 | 反什么偏见 |
|---|------|----------|
| 1 | 甲乙每条提案必含"致命缺陷自述" | 代言人偏见（self-advocacy） |
| 2 | Phase 2 提案随机打乱顺序提交 | 位置偏见（positional） |
| 3 | 每条质疑/回应≤200字 | 长度偏见（length） |
| 4 | 法官 Phase 2 禁止表态 | 权威偏见（authority） |
| 5 | Phase 1 禁止四队互通 | 锚定偏见（anchoring） |
| 6 | 丙队/甲乙立场锁定不可倒戈 | 从众偏见（conformity） |
| 7 | ⭐ **法官不下场替任何一方辩护** | **自我偏好偏见（self-preference）** |

约束 1-6 继承自 `adversarial-architecture-selection`。约束 7 是本技能针对"方案作者裁决自己方案"新增的核心铁律。

---

## 二、约束 7 详解（本技能的灵魂）

**症状**：法官（Claude）同时是被审方案的作者。裁决时本能偏向"保留自己方案根基"的角度，把挑战方案根基的发现轻描淡写驳回。

**为什么危险**：这正是用户要 `pk` 解决的根本问题。如果法官能下场替甲队（自己的决策）辩护，整个对抗结构就退化成"作者自己说了算"，第三方审查的力度全部漏光。

**约束实现**：
- 甲队（Claude 决策辩护）必须由**独立子代理**扮演，法官本人不写甲队立场书
- 法官在 Phase 2 不得以任何形式增援甲队（不补充论据、不暗示倾向）
- 法官在 Phase 3 裁决时，若发现自己倾向某条 Claude 决策，必须明确检查："这是因为幸存论点支持它，还是因为它是我写的？"
- 甲乙提案**对等受审** —— Claude 决策不享有"免审基准"地位

**自检**：Phase 3 裁决前问自己 —— 如果把甲组和乙组的署名对调（假装甲组是别人写的、乙组是我写的），我的裁决会变吗？会变 → 说明 self-preference 在作祟，重裁。

---

## 三、约束 1-6 要点（继承）

**约束1 致命缺陷自述**：甲乙每条提案自报≥1个软肋，带证据。不报视为失职、信用打折。原理：自报永远比被挖出更可控；强制 self-attack 是 Constitutional AI"自我批评"的延伸。

**约束2 随机打乱**：法官 Phase2 启动前打乱所有提案顺序，记录到内部 log。不用 Phase1 完成顺序、不用甲组在前乙组在后。原理：位置偏见可达 30%+，第一条天然占优。

**约束3 ≤200字**：硬上限，超字要求重写。原理：长=详细=可信是 LLM 学到的人类偏见，会让能写长的一方占优。200字够说清1-2个核心点。

**约束4 法官禁言**：Phase2 法官只协调不评价。允许：广播提案、传质疑、提示未回应、宣布进轮。禁止：说"甲队更有道理"、暗示倾向。原理：法官表态后各队会附和（RLHF 取悦权威）。若不慎表态 → Phase3 对该方向降权 + 在裁决表披露。

**约束5 禁互通**：Phase1 四队 mailbox 静默，法官监控，发现互通叫停。原理：即使不看对方立场书，互通也会泄露立场，污染独立性。

**约束6 立场锁定**：丙队不许说"其实还行"，甲乙不许主动撤回整条提案（可承认部分质疑）。倒戈意愿留 Phase3 法官参考。原理：多轮辩论中少数派倾向放弃立场加入多数 = 从众失效，对抗作废。

## 四、法官 Phase 3 裁决前自检清单

```
□ 约束7：如果甲乙署名对调，我的裁决会变吗？（会变 → self-preference，重裁）
□ 我在 Phase 2 中是否表态过？（如有，相关论点降权）
□ 甲乙所有提案是否都含致命缺陷自述？（缺失的信用打折）
□ 我的裁决是否基于"哪条论证更长/更专业"？（如是，重评）
□ 我的裁决是否基于"这条是我写的所以更靠谱"？（如是，回去找客观依据）
□ 被驳回的提案，理由是否引用了 transcript 具体内容？（不能凭印象）
□ 丁队的第三路/隐藏前提/虚假共识预警，是否都在裁决中正面回应了？
□ 存疑未决的提案，是否如实进了裁决表"存疑"栏交用户，而不是我擅自定？
```

任一项不过关 → 回去重裁。
