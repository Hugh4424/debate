# 四阶段对抗裁决协议

> 详细规范五方对抗辩论的 Phase 0 → 1 → 2 → 3 完整流程。法官（Claude）执行时按本规范操作。

---

## Phase 0：立案分派

### 输入
- 本轮审查发现清单（来自 codex/3rd-review）
- Claude 对这些发现的对应决策（采纳倾向/驳回倾向 + 理由）
- decision-log 当前版本号（方案版本锚点，确保各队审的是同一版方案）
- 标准上下文：CLAUDE.md 关键原则（奥卡姆/let-it-crash/karpathy 三铁律）、forbidden files 清单

### 步骤

#### 0.1 拆分提案为甲组/乙组
- **甲组提案** = Claude 在本轮提出/坚持的决策（如"D5 黑名单这么设计"）
- **乙组提案** = codex 本轮提出的发现（如"盲点6 codex 跑挂没降级语义"）
- 同一个争议点若甲乙各有主张，两条都列，分别归组

#### 0.2 写入案卷
路径：`<output-root>/debate/round-<N>/00-案卷.md`（输出根与旧目录兼容规则见 SKILL.md「辩论产物存档目录」）

格式：
```markdown
# debate 案卷 — 第 <N> 轮审查裁决
方案版本锚点：decision-log v<X.Y>
待审提案总数：M（甲组 a 条 + 乙组 b 条）

## 甲组提案（Claude 决策）
- 甲1：{决策内容} | Claude 理由：{...}
- 甲2：...

## 乙组提案（codex 发现）
- 乙1：{发现内容} | codex 证据：{...}
- 乙2：...

## 标准上下文
- 奥卡姆剃刀 / let-it-crash / karpathy 三铁律
- forbidden files：{清单}
```

## Phase 1：独立陈述

### 执行规则

#### 1.1 并行启动
甲乙丙丁四队**同时**启动。法官不指定顺序。

#### 1.2 严禁互通（反锚定偏见）
Phase 1 期间四队之间**禁止**任何 mailbox 通信。法官监控，发现互通立即叫停。每队独立完成立场书。

#### 1.3 各队立场书产出

**甲队**（`position-甲-claude.md`）：对甲组每条决策写强势辩护；**强制致命缺陷自述**——每条决策必须自报至少 1 个最大软肋（带证据）。

**乙队**（`position-乙-codex.md`）：对乙组每条发现写强势辩护，论证该被采纳；同样强制每条自报软肋。

**丙队**（`position-丙-杠精.md`）：对甲乙**所有**提案逐条否定，每条给"不成立/不值得改"的理由 + 证据（引用代码/文档/原则）。禁止凭感觉，禁止和稀泥。

**丁队**（`position-丁-影响搅局.md`）：两部分——
- 中立评估表：对每条提案评 采纳代价(人日/复杂度) / 收益 / 影响半径 / 是否碰 forbidden / 是否违反奥卡姆，给推荐分 1-10
- 搅局清单：甲乙都没想到的第三方案、被"明确不做"排除但值得重提的点、大家默认成立的隐藏前提、可能的虚假共识点

#### 1.4 完成标志
四队写完 idle。全部 idle → Phase 2。

## Phase 2：法庭辩论

### 启动前准备（法官做）
1. 检查甲乙立场书是否含"致命缺陷自述"——缺失退回补全
2. 把所有提案**随机打乱顺序**，记录打乱顺序到内部 log（防位置偏见）
3. 用打乱顺序广播提案列表给四队

### 第一轮辩论
- **丙队**对每条提案发否定质疑（带证据）
- **丁队**对每条提案发 1 条"代价/收益/第三路"挑战（附评分依据）
- **甲队**逐条回应针对甲组提案的质疑（每条≤200字，可承认可反驳可反质疑，禁跳过）
- **乙队**逐条回应针对乙组提案的质疑（同上）

### 共识判断
| 状态 | 处理 |
|------|------|
| 多数提案被驳倒/被坐实，胜负清晰 | 进 Phase 3 |
| 仍胶着、关键提案无定论 | 启动第 2 轮 |

### 第二轮（如需要）
法官列"剩余争议点清单"（≤5），相关队针对争议点再发 1 条质疑/回应。第 2 轮后**强制**进 Phase 3，无第 3 轮。

### 反偏见硬约束（执行时确认，详见 anti-bias-guardrails.md）
- ⚠️ 法官**禁止表态**——只协调、传递质疑、提示"还有N条没回应"、宣布进轮
- ⚠️ 每条回应≤200字（防长度偏见）
- ⚠️ 立场锁定，丙队不许说"这条其实还行"，甲乙不许主动撤回整条提案（可承认部分质疑）

### 输出
`debate-transcript.md`：完整辩论记录，按时间排列，供 Phase 3 引用。

---

## Phase 3：综合裁决

### 3.1 法官单独工作
Phase 3 **不调用任何队**。法官读完所有立场书 + transcript，综合权衡，逐条裁决。

### 3.2 判决方法论：按"幸存论点"加权
- 一条论点被对方驳倒 → 失效，不计入裁决依据
- 一条论点被质疑但成功回应 → 幸存，权重正常
- 一条论点无人质疑 → 幸存，但警惕"是否被忽视"（可能是丙丁漏攻）

**禁止的裁决依据**：哪条论证更长 / 哪条先出现 / 用词更专业 / "我（法官）本来就倾向 Claude 的方案"（self-preference —— 本技能要破的就是这条）

**允许的裁决依据**：丁队可量化的代价/收益评分、对方质疑后无回应或回应空洞、多个独立来源指向同一结论、碰 forbidden/违反奥卡姆的硬事实

### 3.3 逐条裁决
对每条提案给：采纳 / 部分采纳 / 驳回 + 一句决策依据 + 哪些论点幸存哪些被驳。按 [output-template.md](output-template.md) 输出。

### 3.4 过反偏见自检清单
见 [anti-bias-guardrails.md](anti-bias-guardrails.md) 第四节。任一项不过关 → 回去重裁。

---

## 团队清理
Phase 3 写完后：通知四队结束 → 逐个 shutdown → 清理 team 资源 → 把裁决表呈给用户。

## 异常处理
| 异常 | 处理 |
|------|------|
| 某队 Phase1 超时 | 法官介入询问是否遇阻，不催促 |
| 甲/乙立场书缺致命缺陷自述 | 退回补全，不进 Phase 2 |
| 丙队和稀泥（"其实也行"） | 法官纠正，要求回归全否定立场 |
| 甲/乙想撤回整条提案 | 警告立场锁定；撤回意愿留 Phase 3 法官参考 |
| 第2轮后仍胶着 | 强制进 Phase 3，凭幸存论点裁决；拿不准的进裁决表"存疑"栏交用户 |
| codex 审查发现为空 | 无提案可审，不开庭，直接告知用户本轮无发现 |
