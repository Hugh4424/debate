# 裁决结果输出格式

> Phase 3 法官综合裁决的产出格式。分两层：给用户看的"裁决表"（精简）+ 落盘存档的"完整裁决书"（审计用）。

---

## 一、给用户看的裁决表（精简，呈在聊天里）

按"只给最终结果 + 决策依据"原则。每条提案一行：

```markdown
## 第 <N> 轮审查裁决（基于五方对抗辩论，decision-log v<X.Y>）

| 提案 | 来源 | 裁决 | 决策依据（一句） | 幸存论点 → 被驳论点 |
|------|------|------|------------------|---------------------|
| 甲1 黑名单这么设计 | Claude | 部分采纳 | 丁队第三路代价更低，采纳其简化版 | 甲"覆盖全"幸存；甲"必须穷举"被丙驳 |
| 乙1 codex跑挂降级 | codex | 采纳 | 丙丁均无有效反驳，let-it-crash 支持 | 乙全论点幸存 |
| 乙3 某发现 | codex | 驳回 | 丙队坐实"现有机制已覆盖" | 乙论点被代码证据驳倒 |
| 甲4 某决策 | Claude | 存疑 | 甲乙胶着，需用户定 | 见下方存疑说明 |

### 存疑项（交用户拍板）
- 甲4：{为什么裁不了，两方核心分歧是什么，法官倾向 + 不确定的点}

### 一句话总结
本轮 M 条提案：采纳 a 条、部分采纳 b 条、驳回 c 条、存疑 d 条交你定。
```

**关键**：裁决表必须让用户一眼看清每条的归宿和理由，不必翻 transcript。存疑项老实交出去，不擅自定。

---

## 二、落盘存档的完整裁决书（审计用，用户不必翻）

路径：`<output-root>/debate/round-<N>/裁决书.md`（输出根规则见 SKILL.md；读兼容旧版 `pk/` 轮次目录）

```markdown
# debate 裁决书 — 第 <N> 轮
> 方案版本锚点：decision-log v<X.Y>
> 裁决时间：<DATE>
> 依据：本目录下 position-甲-claude.md / 乙-codex.md / 丙-杠精.md / 丁-影响搅局.md + debate-transcript.md

## 1. 逐条裁决（含完整理由）
### 甲1：{提案}
- 裁决：部分采纳
- 幸存论点：{列出，引用 transcript 行号}
- 被驳论点：{列出 + 谁驳的 + 证据}
- 丁队影响评分：{代价/收益/影响半径/碰forbidden/奥卡姆}
- 决策依据：{为什么这么裁，必须基于幸存论点，不得基于 self-preference}

（每条提案同上格式）

## 2. 反偏见自检记录
- 约束7（甲乙署名对调测试）：{结果}
- 法官 Phase2 是否表态：{是/否，如是哪里}
- 其余自检项逐条记录

## 3. 丁队搅局采纳情况
- 第三路：{采纳了哪个/为什么没采纳}
- 拽回的"明确不做"：{结果}
- 隐藏前提：{暴露了什么，怎么处理}
- 虚假共识预警：{有无，怎么处理}

## 4. 回写 decision-log 指引
本轮裁决导致 decision-log 的哪些变更（新增决策/修正/版本号递增到 v<X.Y+1>）
```

---

## 三、裁决后的动作

1. 把裁决表呈给用户（聊天）
2. 完整裁决书 + 四队立场书 + transcript 落盘存档
3. 存疑项请用户拍板
4. 用户拍板后，按"回写 decision-log 指引"更新 decision-log，版本号递增（对齐方案版本锚点 D8）
5. 触发防漏阀检查（D3.1）：若有 codex 发现被驳回、但属 blocking 级，必须在 decision-log 写"codex 反对X/我的决定Y/理由Z"

---

## 四、格式铁律
- 裁决表给人看，用中文、大白话，让用户一眼懂每条归宿
- 决策依据必须可追溯（引用 transcript / 代码 / 原则），禁止"我觉得"
- 存疑不可耻 —— 裁不清的老实交用户，比硬裁错强
