# debate — 决策对抗

## ⚠️ 可选便利层声明

本技能 `debate` 是**可选便利层**，非质量地基：

- **依赖 Agent Teams** 多角色协作能力（`CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1`），不是每个环境都可用的核心能力
- **缺它 intake 仍能完整走完** — quality gate 押在与 AI 宿主无关的关卡（决策记录落盘 + 用户批准）上
- **本技能不阻塞工作流** — 环境不具备时自动降级，用户不必手选

## 环境自动判定

本技能在进入裁决环节前自动检测环境能力，用户不必手选：

| 环境条件 | 执行路径 |
|---------|---------|
| 有 Agent Teams（`CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1`） | 自动启用五方法庭式对抗辩论（见下方完整流程） |
| 无 Agent Teams（单 agent CLI 等环境） | 自动降级单人三档（见降级路径段） |

## 降级路径：单人三档

无 Agent Teams 环境时，不启用五方对抗，改为单人分三档裁决：

- **🟢 必须确认** — 发现明显正确、不改会出问题，裁决「采纳」
- **🟡 建议考虑** — 发现有道理但非必须，裁决「衡量后决定」
- **🔴 仅供参考** — 发现方向偏或当前不适用，裁决「驳回，注明理由」

降级路径的输出格式与五方对抗一致（逐条裁决表），只是不附对抗 transcript。

## 这个技能解决什么

每轮 codex（或其他第三方）审查出结果后，**由谁来裁决哪些发现采纳、哪些废弃**？

默认是 Claude 单人裁。但 Claude 通常是被审方案的作者，会系统性偏向"保留自己方案根基"的角度，把挑战方案根基的发现轻描淡写地驳回 —— 这是 **self-preference bias**，让第三方审查的力度在裁决环节被稀释。

`debate` 把这个裁决换成**五方法庭式对抗**：Claude 的决策和 codex 的决策**对等上庭受审**（谁都不是免审基准），由独立子代理分饰控辩各方互相质疑，Claude 本人只当 Lead 法官 —— **全程禁言、不下场写任何一方的辩护书、只综合幸存论点做最终裁决**。

## 与"禁止自审自判"的边界（重要）

本技能**不替代独立审查**。codex 审查照常由真 codex 独立产出（走 `3rd-review`）。`debate` 只在**审查结果已经出来之后**，对"如何取舍这些发现"做对抗辩论。法官禁言、不下场辩护，是在**加强**裁决的独立性 —— 与"审查报告必须由独立审查员产出"同向，不冲突。

❌ 禁止误用：不得用 `debate` 的子代理去**生成审查发现本身**或**替代 codex 出 verdict**。`debate` 的输入永远是"已经存在的审查发现 + 已经存在的 Claude 决策"，不是凭空造发现。

## 五方结构

| 队 | 角色 | 立场 | 谁来扮 |
|---|------|------|--------|
| **甲队** | Claude 决策代言人 | 论证 Claude 提出的每条决策成立、该这么做 | 独立子代理 |
| **乙队** | codex 决策代言人 | 论证 codex 提出的每条发现成立、该被采纳 | 独立子代理 |
| **丙队** | 杠精 | **所有提案（无论甲乙）都不成立 / 不值得改**，纵向逐条否定，立场锁定不倒戈 | 独立子代理 |
| **丁队** | 影响评估员 + 搅局者 | 中立评每条采纳的代价/收益/影响半径/是否碰 forbidden·违反奥卡姆；横向提甲乙都没想到的第三路、把"明确不做"拽回来质疑、戳隐藏前提、专防虚假共识 | 独立子代理 |
| **Lead** | 法官 | 全程禁言，只综合幸存论点逐条裁决 | Claude 本人 |

**核心铁律**：甲乙对等 —— Claude 的决策不是"基准"，和 codex 发现一样是被告，同样被丙丁攻击。法官（Claude）**不下场辩护**，这是比来源技能多加的一条反偏见约束，专门破 self-preference。

## 粒度

**整轮一场辩论**：一轮 codex 审查的所有发现 + 对应的 Claude 决策，打包进**一场**辩论，不是每条发现单独开庭。

## 四阶段流程（严格不可跳）

```
Phase 0: 立案分派    法官准备案卷（本轮发现 + Claude 决策 + decision-log 当前版本 + 标准上下文），分派四队
Phase 1: 独立陈述    甲乙丙丁并行写立场书，禁止互通；每份强制"致命缺陷自述"
Phase 2: 法庭辩论    提案顺序打乱、每条回应≤200字、1-2轮、法官禁言、立场锁定不倒戈
Phase 3: 综合裁决    法官综合幸存论点逐条裁决（采纳/部分采纳/驳回）+ 依据，过反偏见自检
```

详细规范见 [references/arbitration-protocol.md](references/arbitration-protocol.md)。

## 脱平台目标使用场景

本技能有真实独立外部场景，不依赖任何特定工作流平台：

**独立使用**：给定两份（或多份）方案文本，跑一轮对抗辩论，产出裁决书。例如"对比方案 A 和方案 B，用五方法庭式对抗决定采用哪个"——无需 task 结构、无需 codex 审查前置、无需 decision-log，只需提供方案文本即可启动。

脱平台时输出根按 D18 规则（见下方「辩论产物存档目录」）。

## 辩论产物存档目录（参数化输出根）

**输出根**：
- 有 task-dir 传参 → 以 `{output-root}/` 为根
- 无传参 → 当前会话根目录建 `tasks/{task-name}/`，产物落 `tasks/{task-name}/artifacts/`

新写产物统一使用 `debate/` 子目录：`{output-root}/debate/round-{seq}/`。读取时兼容旧版 `debate/` 之前的 `pk/` 轮次目录（找不到 `debate/` 时回退读取 `pk/` 目录）。

所有辩论产物落盘到输出根下的 `debate/round-{seq}/`，按轮次分子目录，每轮包含：

| 文件 | 内容 |
|------|------|
| `00-案卷.md` | 本轮待审提案清单（审查发现 + Claude 决策） |
| `position-甲-claude.md` | 甲队（Claude 决策代言人）立场书 |
| `position-乙-codex.md` | 乙队（codex/审查发现代言人）立场书 |
| `position-丙-杠精.md` | 丙队（杠精）立场书 |
| `position-丁-影响搅局.md` | 丁队（影响评估+搅局）立场书 |
| `debate-transcript.md` | 辩论 transcript（每轮≤2 轮） |
| `裁决书.md` | 最终裁决书（含决策溯源三字段模板） |

无 Agent Teams 降级单人三档时，留痕目录不变（产物缩为案卷 + 裁决书），留痕不降级。

## 退出标准

debate 辩论的退出由以下条件决定，满足任一条即停：

1. **裁决书产出即停** — 法官完成综合裁决并输出裁决书后，轮次立即结束
2. **累计 2 轮强制停** — 同一场辩论累积 2 轮后，无论裁决是否完成，强制停止
3. **所有提案被裁决完毕** — 案卷中全部提案均已被采纳/部分采纳/驳回/存疑

**D25 达上限行为**：条件 2 触发（达到 2 轮上限），且尚未产出裁决书、仍有未决提案时，必须输出 `unresolved_decision` 清单——列出所有未决提案（含各方论点要点）——并**要求人工裁决**，不静默停止。对应 pk-rules.ts 的 `requiresHumanArbitration: true` 信号，orchestrator 读到此信号后必须将未决清单呈给用户拍板，不可自行关闭。

退出后，由 orchestrator 读取产物判断是否停（不依赖 gate，见 orchestrator 退出判停）。

## 五步执行

### Step 1：判断是否触发（MR-2 方向级分歧判定）

#### 触发标准

debate 触发标准按 **MR-2 四分类**判定：

| 分类 | 含义 | 触发 debate？ |
|------|------|:---------:|
| **问题改变** | 方向级——要解决的问题本身变了（真实痛点 vs agent 发明的问题） | ✅ 触发 |
| **范围/优先级** | 方向级——该不该现在做、scope 边界是否合理 | ✅ 触发 |
| **对需求的解释** | 方向级——原始需求的理解分歧 | ✅ 触发 |
| **仅实现层** | 方案/实现细节争议，不改变方向判断 | ❌ 不触发 |

#### 触发条件

- 存在**至少一条**前三类（方向级）方向级分歧未闭合 → 进入 debate 辩论
- 全部为仅实现层争议 → **不触发** debate，由主 agent 自行处理
- 单条无争议发现不必开庭

#### 触发判断留痕（强制）

触发 debate 的判断不能由主 agent 私下裁量。每次触发或不触发，都必须记录以下结构：

```
### debate 触发判断留痕
- 反对 X：[触发或不触发的具体分歧内容]
- 决定 Y：[进入 debate / 不进入 debate]
- 理由 Z：[依据 MR-2 分类和具体证据]
```

记录写入输出根下的 `debate/trigger-decision.md`。缺此记录 → 视为 debate 触发未执行。

### Step 2：确认前置 + 启用 Agent Teams
- **环境自动判定**：检查 `CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS` 是否启用
  - 已启用 → 继续五方辩论流程
  - 未启用 → 走降级路径（单人三档，见降级路径段）
- 准备好案卷：本轮审查发现清单、Claude 对应决策、decision-log 当前版本号（方案版本锚点）

### Step 3：Phase 0 立案分派
把"待审提案"分成两组：甲组=Claude 的决策，乙组=codex 的发现。写入案卷。

### Step 4：spawn 四队
按 [references/role-spawn-templates.md](references/role-spawn-templates.md) 组装四队 spawn prompt。

### Step 5：执行四阶段 + 裁决
按 [references/arbitration-protocol.md](references/arbitration-protocol.md) 驱动；Phase 2 前读 [references/anti-bias-guardrails.md](references/anti-bias-guardrails.md) 落实反偏见；裁决按 [references/output-template.md](references/output-template.md) 输出。

## 输出（只给最终结果 + 决策依据）

给用户看的是**逐条裁决表**（每条提案：采纳/部分采纳/驳回 + 一句决策依据 + 哪些论点幸存/被驳）。完整辩论 transcript 落盘存档供审计，用户不必翻。用户基于这份更客观的裁决做最终拍板。

裁决导致丢弃任何原始需求台账条目时，必须登记丢弃理由，理由缺失对裁决有强制阻断力。

## 裁决书决策溯源模板

裁决书中每条裁决必须包含以下三字段，确保决策可追溯：

| 字段 | 内容 | 示例 |
|------|------|------|
| **决策** | 本条裁决的具体结论 | 采纳 codex 发现：黑名单改为配置驱动 |
| **依据** | 证据/审查发现编号 + 锚点 | codex-review-002-finding-3，用户原话"不要硬编码"（debate-r1-裁决2） |
| **判定方** | 用户原话·codex 意见·辩论共识 | 辩论共识（甲队论点幸存 + 丁队影响评估支持） |

### 锚点格式

`debate-r&lt;N&gt;-裁决&lt;M&gt;` — 可反查对应的辩论轮次和裁决条目编号。

例如：
- `debate-r1-裁决3` → 第 1 轮 debate 辩论的第 3 条裁决
- `debate-r2-裁决1` → 第 2 轮 debate 辩论的第 1 条裁决

裁决书记录时，每条决策标注锚点。decision-log 的「决策溯源全览」节引用该锚点。

## 不要做的事

- ❌ 法官（Claude）在 Phase 2 表态或下场替甲队辩护 —— 破坏整个技能的目的
- ❌ 把 Claude 决策当成不受审的基准 —— 甲乙必须对等上庭
- ❌ 跳过"致命缺陷自述" —— 反偏见核心机制
- ❌ 按提案原顺序提交给各队 —— 必须打乱（防位置偏见）
- ❌ 超过 2 轮辩论 —— 边际收益骤降
- ❌ 用 `debate` 子代理生成审查发现或替代 codex 审查 —— 越界，撞"禁止自审自判"
- ❌ 无 Agent Teams 环境强制跑五方辩论 —— 必须降级，不可强制开庭
- ❌ 法官在 Phase 0 分派后、Phase 1 结束前偷看任何一队的立场书草稿 —— 破坏独立陈述隔离，等同于锚定偏见
